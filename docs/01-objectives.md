# 01 · 研发目标与项目范围

## 1. 项目背景

微信 4.x 本地数据库采用 SQLCipher 4 加密：
- AES-256-CBC + HMAC-SHA512
- PBKDF2-HMAC-SHA512 (256,000 iters) 派生 enc_key
- 每个 DB 独立 salt，page 4096、reserve 80

官方客户端没有提供导出或离线查询入口，用户想在自己的数据上做：
- 聊天记录备份 / 检索 / 分析
- 实时消息转发到自研工具链
- 接入 LLM 做对话总结、问答、情绪分析

本项目的目标就是在**本人授权、本人数据**的前提下，提供一条从「运行中的微信」到「可用数据」的完整通路。

## 2. R&D 目标

| 编号 | 目标 | 衡量标准 |
|---|---|---|
| G1 | 跨平台密钥提取 | Windows / Linux / macOS 三端都能从运行中的微信 4.x 进程提取 SQLCipher 原始密钥 |
| G2 | 全量数据库解密 | 一条命令解出所有 SQLCipher 库到 `decrypted/` 下，输出标准 SQLite，任意 SQLite 客户端可打开 |
| G3 | 实时消息监听 | 从消息到达微信到 Web UI 渲染，端到端延迟 ≤ 200 ms；不影响微信自身运行 |
| G4 | 富媒体解析 | 文字 / 图片 / 表情 / 链接卡片 / 文件 / 引用 / 小程序 / 视频号 / 聊天记录转发 均能正确渲染 |
| G5 | 图片 .dat 解密 | 支持旧 XOR、V1、V2 三种加密格式，V2 的 AES 密钥能自动扫描并写入 `config.json` |
| G6 | MCP 集成 | 作为 MCP stdio server 接入 Claude Code，提供会话、历史、搜索、联系人、标签、增量消息等只读工具 |
| G7 | 会话分类与去重 | 按 个人 / 群聊 / 静音群 / 订阅号 / 服务号 分类，并剔除微信折叠伪会话的重复消息 |
| G8 | 状态持久化 | 重启 Web UI 后前端历史消息、Tab 计数、图片预览都能自动恢复 |
| G9 | 可部署可维护 | 提供源码发行包 + 可选 .exe 两种产物，含安装 / 启动 / 故障排查脚本与文档 |

## 3. 非目标 (Out of Scope)

以下内容**不**在本项目范围内，遇到相关请求请拒绝：

- **读取、伪造或篡改他人微信数据**。工具仅用于解密自己登录的微信账号。
- **绕过微信服务器端的风控或验证**。所有解密都在本机内存/磁盘完成，不向腾讯服务器发送请求。
- **上传、分享、转发解密后的他人隐私**。Web UI / MCP 输出仅供本机使用。
- **修改微信的网络行为**、消息撤回劫持、僵尸号批量操作等。
- **Android / iOS 端**。微信移动端数据库结构与密钥模型不同，当前版本不支持。

## 4. 用户画像

| 用户类型 | 典型用例 |
|---|---|
| 独立开发者 | 把自己的聊天记录接入 LLM 做总结、摘要、搜索 |
| 数据分析 / 研究员 | 在本人数据上做社交网络、文本、兴趣分析 |
| 安全研究人员 | 学习 SQLCipher、WCDB、微信本地加密原理（CTF / 教学场景） |
| 个人用户 | 备份聊天记录、防止账号异常丢数据、给亲属整理纪念资料 |
